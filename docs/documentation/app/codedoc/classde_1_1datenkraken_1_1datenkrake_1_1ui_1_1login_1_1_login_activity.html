Detailed Description

The login activity is used as a gateway and the first activity, that is called, when opening the app. This will ensure that a user has to login to proceed with a valid AccessToken. The Token is also refreshed, when the app is opened.

Uses AuthenticationManager#updateAuthStateWithAuthorizationResponse(AuthorizationResponse, AuthorizationException) to update the AuthenticationManager. If the response is not null, it will get a TokenRequest and call AuthorizationService#performTokenRequest(TokenRequest, AuthorizationService.TokenResponseCallback) with it. ◆ onCreate()

void de.datenkraken.datenkrake.ui.login.LoginActivity.onCreate ( Bundle  savedInstanceState)

Called, when the Activity is created.
Retrieves the SharedPreferences from the LOGIN_STORAGE and, if ACCEPTED_DATA_COLLECTION is false, it will display a DataCollectionPopupFragment that the user has to accept to proceed.
Also creates a new AuthenticationManager and AuthorizationService, and checks, if the user is authorized. If this is the case, it redirects to the MainActivity, else it calls LoginActivity#setupUI().

Start the authorization flow.
Gets an AuthorizationRequest from the AuthenticationManager and creates an intent with it through AuthorizationService#getAuthorizationRequestIntent(AuthorizationRequest).
Starts an activity waiting for a result, with the request code ACTIVITY_RESULT_AUTH.
This has to be called on another thread.
